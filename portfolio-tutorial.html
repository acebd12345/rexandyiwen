01 / 05

設計
哲學

大多數工程師的作品集有個矛盾：你有能力打造複雜系統，卻把自己的網站弄得比維護客戶案子還麻煩。

重點不是技術有多複雜，而是資料與設計是否真正分離——改一行文案，不該重新部署整個前端。

這套架構的核心哲學：把 Google Sheets 當資料庫，Apps Script 當 REST API，靜態 HTML 當渲染引擎。三層各司其職，互不耦合。

01
資料與設計分離
設計師改試算表，工程師改 HTML，兩者互不影響。唯一的依賴是 API 格式的穩定。
02
三層 Fallback 容錯
GAS 掛掉 → 讀本地 JSON；JSON 不存在 → 用內嵌 FALLBACK。網站永遠不崩潰。
03
免費但有韌性
GCS 靜態託管 + Apps Script 免費方案足夠個人作品集。成本趨近零，SLA 趨近 Google 等級。

04 / 05

Apps
Script

只需要一個 doGet(e) 函式。接收 ?table= 查詢參數，從對應分頁讀出所有列，轉成 JSON 回傳。整個 API 大約 30 行。

Code.gs — Apps Script 
// 部署為 Web App → 任何人可存取
function doGet(e) {
  const table = e.parameter.table;
  const ss    = SpreadsheetApp.getActiveSpreadsheet();
  const sheet = ss.getSheetByName(table);

  if (!sheet) {
    return jsonOut({ error: `Sheet "${table}" not found` });
  }

  const [headers, ...rows] = sheet.getDataRange().getValues();
  const data = rows
    .filter(row => row.some(c => c !== ""))  // 過濾空列
    .map(row => {
      const obj = {};
      headers.forEach((h, i) => obj[h] = row[i]);
      return obj;
    });

  return jsonOut(data);
}

function jsonOut(data) {
  return ContentService
    .createTextOutput(JSON.stringify(data))
    .setMimeType(ContentService.MimeType.JSON);
}

日常
更新

上線後，日常維護不需要碰任何程式碼。所有內容更新都在 Google Sheets 完成，網站自動反映最新資料。

STEP 1
打開 Google Sheets
找到對應的分頁——新增一列 note、修改 project 描述、更新相片 EXIF 資訊。就像操作 Excel 一樣。
rex-chang-notes rex-chang-projects rex-chang-albums
STEP 2
儲存 — 完成
不需要重新部署。下一個訪客打開網站時，Apps Script 會即時從 Sheets 讀取最新資料，前端動態渲染。
即時生效 零部署 零 downtime
FALLBACK 機制
三層容錯，永不崩潰
GAS 回應失敗 → 改讀 rex-chang-content.json；JSON 不存在 → 讀 HTML 內嵌的 FALLBACK 物件。無論如何，網站都能正常顯示完整內容。
GAS → local JSON → FALLBACK const
HOSTING
靜態檔案任意部署
整個網站是一個 HTML 檔案。上傳到 GCS Bucket 並開啟 public access，或丟上 GitHub Pages、Cloudflare Pages——三種方式都免費，全球 CDN。
